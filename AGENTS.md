# Project Instructions

Before committing changes in this repository:

1. Determine whether any new tests should be created. If so, create them.
2. Run ESLint. Address any findings. If ESLint is not configured or unavailable, report that explicitly.
3. Run `coderabbit`. Address any valid findings. Report any invalid findings.
4. Ensure all tests pass. If tests are no longer valid, report them explicitly.
5. Run `trufflehog` to scan for secrets.
6. Use the `notify` command to tell the user when done, or when human input is needed.
